## Commit messages
- NEVER include "generated by AI", "co-authored-by" AI references, or any AI tool mentions
- NEVER include time spent, next tasks, or TODO lists in commit messages
- Focus on *what* changed and *why* — not how it was written
- Use imperative mood ("Add feature" not "Added feature")
- Keep the subject line under 72 characters
- Add a body only when the *why* isn't obvious from the subject

## PR descriptions
- No AI tool names, URLs, or badges in PR titles or descriptions
- Describe the change from a developer's perspective, not a tool's
- Focus on the problem being solved and the approach taken

## Code
- No "generated by AI" or similar comments in source files
- No AI attribution in file headers, docstrings, or inline comments
- Write code as if a human wrote it — because a human directed it
