# Claude Code Plugins

Shared skills for Claude Code — cost optimization, clean workflows, and productivity.

## Plugins

| Plugin | Type | Description |
|--------|------|-------------|
| **cost-optimization** | auto-active | Reduce token usage by 30-60% |
| **clean-commits** | auto-active | No AI fingerprints in git history |
| **docs-sync** | auto-active | Keep local docs in sync with code changes |
| **slack-message** | `/slack-message` | Format and copy Slack-ready messages |
| **laravel-docker-switch** | `/laravel-docker-switch` | Toggle Laravel between local and Docker |

### cost-optimization
Automatically routes subagent tasks to cheaper models, prefers Edit over Write (diffs vs full files), and keeps responses concise. No configuration needed — just install and save tokens.

### clean-commits
Enforces clean git practices: no "generated by AI" comments, no co-authored-by AI lines, no task lists in commits. Commit messages stay focused on *what* and *why*. Works silently in the background.

### docs-sync
Watches for code changes that affect project structure, APIs, or workflows and updates your local `docs/` folder accordingly. Never commits docs unless you ask — designed for teams that keep documentation local.

### slack-message
Invoke with `/slack-message` to convert any message into proper Slack markup. Handles bold, italic, bullets, code blocks, emoji, and blockquotes. Copies to clipboard — just Cmd+V into Slack.

### laravel-docker-switch
Invoke with `/laravel-docker-switch local` or `/laravel-docker-switch docker` to toggle your Laravel app between running on the host (with Docker services) and running fully inside Docker. Reads your `docker-compose.yml` to determine service names automatically.

## Install all plugins

Add to `~/.claude/settings.json`:

```json
{
  "skills": {
    "claude-plugins": {
      "source": "github:VorobiovD/claude-plugins"
    }
  }
}
```

## Install individual plugins

```json
{
  "skills": {
    "cost-optimization": {
      "source": "github:VorobiovD/claude-plugins/plugins/cost-optimization"
    }
  }
}
```

Replace `cost-optimization` with any plugin name from the table above.

## Known issue: slash commands not auto-discovered

Claude Code currently has a bug where user-invocable skills from community marketplaces don't appear in the skill discovery system ([#15178](https://github.com/anthropics/claude-code/issues/15178)). Auto-active plugins (`cost-optimization`, `clean-commits`, `docs-sync`) work fine. Slash commands (`/slack-message`, `/laravel-docker-switch`) may not be found automatically.

**Workarounds:**

- Type `/slack-message` or `/laravel-docker-switch` directly — manual invocation still works
- Copy skill files to `~/.claude/skills/` for reliable discovery:
  ```
  mkdir -p ~/.claude/skills/slack-message ~/.claude/skills/laravel-docker-switch
  cp plugins/slack-message/skills/slack-message/SKILL.md ~/.claude/skills/slack-message/
  cp plugins/laravel-docker-switch/skills/laravel-docker-switch/SKILL.md ~/.claude/skills/laravel-docker-switch/
  ```
- Clear plugin cache if skills aren't loading: `rm -rf ~/.claude/plugins/cache`

## Clipboard permissions (for slack-message)

Add to `~/.claude/settings.json` under `permissions.allow`:

```json
"Bash(printf*pbcopy)",
"Bash(*pbcopy*)",
"Bash(pbcopy*)"
```
